Application.java

Der "Zauber" steckt in der Annotation.

@SpringBootApplication
class Application(){...}

Start der Web-Anwendung

SpringApplication.run(Application.class, args);

"println" auf dem Server

Initialisieren:

static final Logger log = LoggerFactory.getLogger(Application.class);

Benutzen:

log.info("Hallo, Welt!");

Alternative Kanäle: debug, warn, error

Stereotypen der App-Komponenten

@Component beschreibt eine "arbeitende" Klasse.

3 Spezialisierungen:

@Controller (bzw. RestController)
für eine Webschnittstelle.

Repository für eine persistente Modell-Sammlung

@Service für einen internen, zustandslosen Service

Cronjobs einrichten

Annotation vor der Methode, die regelmäßig ausgeführt werden soll.

@Scheduled(fixedRate = 5000)
public void currentTime(){ ... }

Erfordert in der Application.java eine "Genehmigung"

@EnableScheduling
class Application(){ ... }

Cronjobs einrichten

Auch komplexere Zeit-Schemata sind möglich

@Scheduled(cron = "Sec, Min, Std, M-Tag, Mon, W-Tag")

Beispiel: Führe den Task stündlich zu jeder 15. Minute zwischen 9 bis 18 Uhr aus und zwar nur von Montag bis Freitag

@Scheduled(cron = "0 15 9-17 * * MON-FRI")

Welcher Cronjob erfüllt die gewünschten Kriterien im folgenden Satz?

Alle 10 Sekunden am 29. Februar stündlich von der 20. bis 42. Minute.

10 20-42 * 29 2 *
*/10 20-42 * * 2 29
10 20-41 * * 2 29
*/10 20-41 * 29 2 *

Web-Schnittstelle

Annotation an der Klasse:

@Controller
class GreetingsController() { ... }

Eine Schnittstelle entspricht eine Methode.

@RequestMapping("/hello")
public String greet(Model model){return "template_name"}

Dynamische Website wird aus "/templates/template_name.html" eingebunden

Web-Schnittstelle HTTP Methode

@RequestMapping("/path") hört auf GET-Requests

Lässt sich auch umbiegen auf
POST, PUT, DELETE

@RequestMapping(value="/path",
   method=RequestMethod.POST)

Web-Schnittstelle Parameter

URL-Queries kann man auf die Parameter der Methode mappen:

public String greet(
   @RequestParam(value="name", defaultValue="World")
   String name)

Spezialisierung @RestController

Simpler Text:

public String greet(){return "hello";}

JSON aus Java-Klasse:

public Greeting greet(){return new Greeting();}

Achtung! @Controller und @RestController nicht verwechseln.

Repositories einrichten

Für die Abspeicherung von Objekten

interface MyRepository extends Repository<Person, Long>
{ ... }

Nur Deklarierung der benötigten Methoden.

void save(T entity);
List findAll();
T findByFirstname(String firstname);

Repositories einrichten

Benötigt wird eine Entity + Identifier:

@Entity
public class Person{ ... }

Der Identifier wird mit @Id gekennzeichnet.

@Id
private long id;

Repositories einrichten

Repository im Code initialisieren:

@Autowired
MyRepository repo;

Methoden vom Interface anwenden.

repo.save(new Person("Peter", "Lustig"));
List<Person> person_list = repo.findAll();
Person mr_lustig = repo.findByFirstname("Peter");

Spring weiß durch Namenskonvention, was getan werden muss. "Dependency Injection"

Beispiel "votequiz"

Websocket Controller

Ermöglicht schnellen TCP-Datenaustausch zwischen Client und Server:

@MessageMapping("/candidate")
@SendTo("/channel/all")
public ServerMessage getVote(CandidateVote vote){ ... }

Erfordert Deklaration der Schnittstellen.

siehe WebSocketConfig.java

Websocket Controller

Client-seitig realisierbar mit SockJS & STOMP:

new SockJS('/candidate')
.subscribe('/channel/all', function(json_message){...}
stompClient.send("/quiz/candidate", {}, json_message)
Sende-Kanal: quiz/candidate
Empfänger-Kanal: channel/all
